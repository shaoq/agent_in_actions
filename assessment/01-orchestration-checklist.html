编排控制层模块考核

目标

验证学习者是否真正具备编排控制层的理解、实现、调试和面试表达能力。资产文件完成不等于模块已掌握;只有提交证据并达到通过标准后,才能更新掌握状态。

考核前应完成:

  • 编排控制专题
  • 最小 Agent Loop 实验
  • 编排面试题

总体规则

  • 总分:100 分。
  • 建议总时限:120 分钟。
  • 通过线:总分不低于 80 分。
  • 单项门槛:理解、实现、调试、表达每项得分不低于该项满分的 60%。
  • 必须提交四类证据;缺少任一类证据均不判定为已掌握。
  • 首次未通过时记录具体缺口,补强后使用新案例复测,不直接修改原评分。

任务一:架构理解

分值与时限:25 分,20 分钟。

输入:一张空白纸或空白 Markdown,不得查看专题文档。

任务:

  1. 画出单 Agent 的 decide -> act -> observe -> terminate 请求链路。
  2. 标出编排层、推理内核、动作层和治理层的职责边界。
  3. 解释运行状态、会话状态和长期记忆的区别。
  4. 给出至少三种终止条件和两种不应该使用 Agent loop 的场景。

评分锚点:

分数 标准
0-9 只能描述 LLM 调用或框架 API,无法说明状态和终止。
10-15 能画出主循环,但边界、失败或状态解释不完整。
16-20 主链路、状态、边界和终止基本正确,能指出常见错误。
21-25 能补充适用前提、替代方案、恢复和治理取舍,表达无明显概念混淆。

通过证据:架构图或 Markdown 回答,附完成时间和自评盲点。

任务二:工程实现

分值与时限:25 分,35 分钟。

输入:labs/agent-loop-minimal/ 现有代码,允许查看 Python 标准库文档。

任务:

  1. 增加一个 cancelled 终止状态和对应 trace event。
  2. 设计一个不依赖全局变量的取消信号接口。
  3. 增加正常取消的自动化测试,保证原有四个测试继续通过。
  4. 说明如果取消发生在不可逆动作执行中,当前实现还缺少什么。

评分锚点:

分数 标准
0-9 代码不能运行,或通过破坏已有行为实现取消。
10-15 能增加状态,但接口耦合、trace 或测试不完整。
16-20 状态、接口、trace 和回归测试完整,职责基本清楚。
21-25 同时说明动作边界、竞态、幂等或 checkpoint 影响,实现保持最小且可扩展。

通过证据:代码差异、完整测试输出、设计说明。考核修改应在独立分支或临时副本完成,不直接覆盖基准实验。

任务三:Trace 调试

分值与时限:30 分,35 分钟。

输入:分别运行以下三个场景产生的 trace:

conda run -n agent_in_actions python labs/agent-loop-minimal/demo.py --scenario max-steps
conda run -n agent_in_actions python labs/agent-loop-minimal/demo.py --scenario invalid
conda run -n agent_in_actions python labs/agent-loop-minimal/demo.py --scenario action-error

任务:

对每条 trace 回答:

  1. 失败或终止发生在哪个 phase?
  2. 已知事实和推测原因分别是什么?
  3. 修复应落在决策契约、编排策略、动作实现还是治理策略?
  4. 修复后需要增加什么回归用例?

再分析一个附加问题:如果 trace 显示 run_completed,但用户任务实际上未完成,为什么当前事件不足以证明业务成功?应该增加什么验收信号?

评分锚点:

分数 标准
0-11 主要根据错误文案猜测,不能使用 phase 和状态证据。
12-18 能定位三种场景,但事实、推测和修复层次混合。
19-24 能基于事件链归因,修复位置和回归用例基本合理。
25-30 能识别 trace 的证据边界,补充业务验收、错误分类和观测字段。

通过证据:三份结构化调试记录,每份包含“现象、证据、归因、修复、回归”。

任务四:面试表达

分值与时限:20 分,30 分钟。

输入:随机抽取编排面试题中的两题,再回答一道系统设计追问:

设计一个可暂停、可恢复的企业工单 Agent 编排方案。说明状态、终止、动作失败、权限、人在回路和 trace。

任务:

  • 每道基础题先用 30 秒回答,再接受 3 分钟追问。
  • 系统设计题在 8 分钟内完成结构化回答。
  • 至少引用一个实验中的失败案例,但不能照读题库答案。

评分锚点:

分数 标准
0-7 回答以术语或框架名为主,缺少链路和取舍。
8-12 能解释主流程,但追问下边界、失败或证据不足。
13-16 按问题、架构、链路、失败、治理、取舍组织回答。
17-20 能结合实验与业务风险,主动说明替代方案和未解决问题。

通过证据:录音、逐字稿或面试官评分表,附一个回答得不好的追问及改进版本。
