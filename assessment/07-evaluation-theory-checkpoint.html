统一场景

你要评估一个多租户企业采购 Agent 的模型、prompt 和编排升级。系统检索合同与供应商材料，创建采购单并发邮件；单笔超过 5 万元需审批。已知风险包括跨租户检索、间接 Prompt Injection、审批后参数变化、错误重试、PII 外发、Judge 偏好冗长答案，以及线上少量人工接管。请设计从目标数据集到受控发布的完整评测方案。

证据要求

  1. 一份带 dataset_version、来源、风险、provenance、切片和保留策略的数据卡，以及候选/基线版本矩阵。
  2. 一条失败 run 的最终结果、至少十个 trace 字段、首个偏离点、传播链和组件 owner；安全 hard fail 不得被总分抵消。
  3. 一份至少四维、带等级锚点的 rubric；规则断言、人工金标、Judge 换位复判、低置信升级与校准指标齐备。
  4. 一个可机器判定的门禁：硬约束、关键切片、总体非劣、成本/延迟、灰度 SLO 与回滚条件。
  5. 一张安全闭环图：隔离红队和脱敏线上 trace 进入离线回归，结论反馈编排/治理/工具等 owner；在线治理仍负责实时拦截。

作答模板

标题：评测层理论检查点作答；A 目标与数据：目标/禁止行为、failure taxonomy、来源/切片、版本/切分、基线/方差；B 结果与轨迹：outcome、trajectory、trace schema、hard fail、首个偏离点→传播链→owner；C 评分者：规则、人工 rubric/金标、Judge prompt/version、换位、校准、分歧升级；D 回归门禁：候选/基线、配对重复、效应与区间、硬约束/切片/总体、灰度/回滚；E 安全反馈：威胁模型、隔离红队、线上脱敏采样、样本治理、归因/修复/复测；边界声明：评测层离线消费结果与 trace 并反馈编排，治理层在线实时控制；评分记录：A/B/C/D/E、总分、单项 60%、证据缺口、补学页、复测日期。
