Agent 能力考核标准

本文件定义全局能力等级。可直接执行的主题考核:

  • 00 Agent 基础理论检查点
  • L1 九层 Agent 架构知识地图检查点
  • 01 编排无代码理论检查点
  • 02 推理内核理论检查点
  • 03 工具 / 动作层理论检查点
  • 01 编排控制层模块考核

L1 九层架构检查点采用 100 分制,总分不低于 80 且五项任务均达到各自满分的 60% 才通过。通过表示能够将系统职责放到正确层级,不表示已经掌握各组件内部实现。

L2 推理内核检查点同样采用双门槛：总分不低于 80，且五项任务每项不低于 12/20。通过表示理论边界与决策设计达到门禁，不表示已完成 L3 工程验证。

L2 工具 / 动作层检查点采用五项各 20 分的双门槛：总分不低于 80，且每项不低于 12/20。通过表示能设计受控工具契约、授权与副作用恢复，不表示已实现生产工具运行时。

评分维度

1. 架构理解

  • 1 分: 只能描述 LLM 调用流程。
  • 3 分: 能说出 Agent 的主要组件。
  • 5 分: 能清楚说明组件边界、依赖关系和常见错误设计。

2. 工程实现

  • 1 分: 只能跑通简单 demo。
  • 3 分: 能实现工具调用、RAG 或记忆中的单项能力。
  • 5 分: 能实现带失败处理、权限、日志和评测的端到端 Agent。

3. 调试能力

  • 1 分: 主要靠改 prompt 试错。
  • 3 分: 能根据日志定位大致问题。
  • 5 分: 能基于 trace 区分模型、检索、工具、状态、权限、评测问题。

4. 安全与治理

  • 1 分: 只知道 prompt injection 这个概念。
  • 3 分: 能设计基础权限和审计。
  • 5 分: 能覆盖在线防护、离线安全评测和沙箱隔离。

5. 面试表达

  • 1 分: 回答零散,缺少结构。
  • 3 分: 能按模块回答常见问题。
  • 5 分: 能结合项目经验、取舍、失败案例和上线策略回答追问。

阶段考核任务

  1. 设计一个客服 Agent,要求覆盖 RAG、工具调用、权限、安全、评测。
  2. 给定一段失败 trace,判断问题来源并给出修复方案。
  3. 实现一个最小可运行 Agent demo,包含至少一个工具和一个评测用例。
  4. 解释 Memory 与 RAG 的边界,并设计防污染策略。
  5. 设计一套 Agent 上线前评测与上线后观测方案。
