通过规则

双门槛 · 满分 100；总分必须 ≥ 80，且任务 A–E 每项必须 ≥ 12/20（该项 60%）。总分达标但任一单项低于 12，仍判定未通过。

  • 必须针对场景提交图、契约、策略与 trace 证据。

统一场景

设计一个多租户旅行助理。它需要记住用户确认过的座位、饮食和预算偏好，保留行程事件，并复用成功的改签程序；企业旅行政策来自外部知识 / RAG。用户可以查看、更正和删除记忆。攻击者会在网页、工具 observation 和对话中诱导系统保存恶意指令或他人信息。

证据要求

  1. 一张四对象边界图，明确“经历记忆 ≠ 外部知识 / RAG”。
  2. 情景、语义和程序记忆契约，包含 provenance、subject、scope、confidence、consent、retention。
  3. 一次冲突事实的候选、确认、整合、读取 trace，记录 policy_version。
  4. 一次用户删除请求传播到主存、索引、缓存、派生摘要和备份策略的证据。
  5. 覆盖跨租户泄露、间接注入、自我回写和敏感信息过度保存。
