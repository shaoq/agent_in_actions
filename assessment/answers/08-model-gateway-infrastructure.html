A · 抽象、路由与能力

完整示范答案

统一请求/响应/错误契约，能力目录记录上下文、模态、结构化输出、区域、合规、价格和 revision。路由先按硬约束过滤，再按质量、延迟、成本和健康度软排序；同一 Run 可保持粘性，路由记录候选、理由和策略版本，模型准入需离线评测。

{
  "request":{"run_id":"R8","purpose":"contract_review","region":"cn-east",
    "requirements":{"context_tokens":120000,"structured_output":true,"tool_calling":true,"data_residency":"CN"},
    "budget":{"deadline_ms":18000,"max_cost_cny":0.6},"route_policy":"rp-v6"},
  "stream":["accepted","message.delta","tool_call.delta","usage","completed|error"],
  "response":{"model_revision":"m-long-202607","finish_reason":"stop","usage":{},"route_reason":"hard filters + best quality"},
  "errors":["invalid_request","no_capable_route","rate_limited","provider_timeout","stream_interrupted","budget_exhausted"]
}

长合同审查先硬过滤 CN 区域、120k context、结构化输出和合规 revision，候选 m-long 与 m-premium，再按离线质量、p95、成本和健康度选 m-long；低延迟分类硬过滤 CN/JSON 后选择 m-fast。路由记录被过滤候选及原因、能力目录版本和 run 粘性；模型 revision 未通过契约/安全/质量准入评测不能上线。推理只面向统一契约，不直连厂商。

关键得分证据

两类请求路由表、能力硬过滤、revision、路由理由和重放证据。

达标与优秀

达标：统一契约、能力目录和路由正确。优秀：准入门禁、粘性、漂移和回滚。

可接受变体

业务可提供质量/成本偏好，但不能指定绕过合规硬约束的厂商。

常见扣分

业务直连厂商；只按价格随机；记录模型名但无 revision。

补学与复测

用高隐私与长上下文两类请求重做硬过滤和软排序。

B · 降级、限流与缓存

完整示范答案

按超时、限流、供应商故障、能力缺失、部分流式输出分类；统一 deadline 和重试预算，隔舱、熔断、half-open 探测。降级前重新验证能力与安全，不让弱模型越权。缓存键包含租户、身份/权限摘要、Prompt/模型/工具版本和数据版本，敏感响应默认不跨主体复用，并支持删除传播。

故障处置
队列拥塞、尚未派发按优先级排队或 load shedding；不消耗 provider 重试
429统一预算内退避一次，隔舱计数；持续超限打开熔断
首 token 超时deadline 有余量才切同能力 route，保留同一 request_id
部分流式输出后中断标记 partial，不拼接另一模型文本；撤回草稿或重新开始并告知用户
区域故障仅切满足数据驻留的同域区域；无安全 route 则 fail closed
预算耗尽/能力缺失停止或降级为只读/人工；禁止弱模型承担工具调用

按租户、优先级和模型池分层限流，provider 隔舱与 circuit breaker 进入 half-open 时只放少量探测。缓存键为 tenant + subject_scope_hash + normalized_request + prompt/model/tool/index/policy versions，TTL 取数据最短有效期；含 PII、工具提议和高风险结果默认不缓存。权限撤回或源数据删除时按血缘清除，记录 hit/miss、key version 和 deletion receipt。

关键得分证据

故障矩阵、分层限流、熔断状态、缓存安全键和命中审计。

达标与优秀

达标：deadline、预算、隔舱、能力复验、缓存隔离。优秀：partial output、unknown、half-open 和撤权传播。

可接受变体

可不缓存生成结果，只缓存受权检索或模型元数据；需说明一致性与成本权衡。

常见扣分

多层各自无限重试；降级静默改变能力；缓存键遗漏租户/版本。

补学与复测

演练流式中断和供应商限流，验证预算、降级、缓存与用户终态。

C · 成本、延迟与容量

完整示范答案

SLO 分解排队、网关、TTFT、生成、工具与提交，关注 p95/p99 而非平均；成本按 Run/成功任务归因 token、模型、工具、存储和重试。容量由 arrival rate × service time 估并受并发、RPM/TPM、上下文和突发限制，保留余量；过载时拒绝低优先、缩减上下文、降级或 load shedding。

峰值并发基线 = 20 RPS × 8s 平均服务时间 × 1.5 突发系数 = 240
加 25% headroom：目标可用并发 = 300；按华东/华南 60/40 配置 180/120，
任一区域故障时不能假定另一域全接管，预留关键流量 120 并对低优先 load shed。
若平均输入 6k、输出 1k token：峰值 TPM≈20×60×7k×1.5=12.6M，
分别核对 provider RPM≥1800、TPM≥12.6M、长上下文并发和输出长度 p95。

端到端 SLO 例如 p95≤18s、p99≤30s，其中排队 2s、网关 0.2s、TTFT 3s、生成 10s、提交/缓冲 2.8s；同时监测 TTFT/TPOT。成本事件按 run 记录 input/output/cache tokens、价格版本、模型、重试、工具和存储，计算单位成功成本而非每 token 单价。队列>0.8 容量或预计 deadline 超限时，先拒绝离线低优先、缩减非关键 context，再选择等能力低延迟模型；不能通过无界排队掩盖过载。

关键得分证据

分位数、并发/TPM 计算、单位成功成本、突发压测和扩缩阈值。

达标与优秀

达标：SLO、成本归因、容量和过载策略。优秀：队列模型、价格版本、分级负载和演练。

可接受变体

批处理可牺牲延迟换吞吐，但仍需 deadline、队列上限和取消。

常见扣分

只报平均 QPS；成本只算 token；扩容不考虑 provider 配额。

补学与复测

用一组峰值流量计算并发、TPM、余量和 load shedding 门槛。

D · 运行时、存储与网络

完整示范答案

计算节点无状态化，Run/Checkpoint 持久到 durable store；消息至少一次投递配合幂等消费者和 outbox/receipt。对象、审计、向量与事务数据按一致性和保留要求分开；工作负载身份访问短期 secret，模型 egress 与工具/沙箱 egress 分离，默认拒绝。

一致性约束：State 与 outbox 原子写，至少一次投递的 consumer 以 event_id/call_id 幂等，effect receipt 回写权威状态；示例目标为 RPO<1m、RTO<15m。

运行容器丢失不会丢 State；cache 可重建且不作权威事实。服务使用 workload identity 换短期 secret，不能挂载长期云凭据。故障演练包括区域断网、队列重复、DB failover、对象恢复和 provider egress 阻断；备份恢复后重放 tombstone/审批失效记录。RPO/RTO、outbox lag、重复消费去重和恢复 run acceptance 都形成证据。

关键得分证据

部署拓扑、状态 owner、消息恢复、网络策略、RPO/RTO 和备份恢复演练。

达标与优秀

达标：durable state、幂等消息、存储和身份/egress 完整。优秀：区域故障、灾备演练和依赖隔舱。

可接受变体

单体可共享运行时，但状态不能只在进程内，网络与 secret 权限仍按工作负载划分。

常见扣分

容器重启丢状态；exactly-once 口号替代幂等；模型和工具共用任意 egress。

补学与复测

演练编排节点在动作后崩溃，证明可恢复且不重复副作用。

E · 沙箱、租户与隔离

完整示范答案

隔离覆盖身份、计算、文件、网络、secret、资源配额和销毁。治理决定允许什么，工具执行器遵守能力契约，基础设施用进程/VM、mount、seccomp、网络策略、短期 secret 和配额强制。每次任务创建干净环境，结束撤销凭据、清理文件并生成销毁 receipt。

{
  "execution_id":"E77","tenant":"acme","run_id":"R8","image":"python@sha256:11",
  "policy":"sandbox-p9","mounts":[{"file":"contract.csv","mode":"ro"}],
  "network":{"default":"deny","allow":[]},"secrets":[],
  "quota":{"cpu":"1","memory_mb":512,"disk_mb":256,"pids":64},"deadline_s":30
}

负向验证：tenant B 文件路径返回 deny；访问 169.254.169.254 和公网失败；../ 路径穿越被 canonical path/mount 边界拒绝；环境/文件中无 cloud secret；fork bomb/内存耗尽被 quota 终止；结束后文件、进程、网络 namespace 和短期身份撤销，重新创建同 execution ID 也看不到残留。治理只决定 Python 分析是否允许，工具定义输入输出，基础设施提供 VM/进程、只读 mount、seccomp 和网络隔离；审批不能替代沙箱。

关键得分证据

跨租户、逃逸、secret 读取、网络外发、资源耗尽和残留负测。

达标与优秀

达标：七维边界和生命周期完整。优秀：镜像/策略证明、逃逸演练、残留扫描和事件响应。

可接受变体

低风险只读任务可用较轻隔离，但必须由风险策略决定且不能共享租户 secret。

常见扣分

只写 Docker；审批替代沙箱；任务结束不销毁凭据和文件。

补学与复测

执行五类负测并保存策略、拒绝事件和销毁 receipt。
