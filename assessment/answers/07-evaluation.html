A · 目标、数据集与基线

完整示范答案

先定义业务成功、hard fail、失败分类和成本/延迟约束，再建含来源、同意、快照、版本、切片和去重的数据卡。按实体/时间防泄漏切分，保留隐藏挑战集；基线至少含普通代码/Workflow、旧版本和简单模型，并重复运行报告方差。

Eval contract eval-purchase-v4
成功：采购单字段正确、审批与业务验收通过、邮件送达正确收件人
hard fail：跨租户证据/PII 外发、无审批高额创建、重复采购、注入控制动作
软目标：任务成功率≥92%，p95≤18s，单位成功成本≤¥0.45，人工接管≤8%
数据卡：dataset=ds-2026-07-19，corpus=idx-v12，policy=p17，来源=脱敏历史+合成红队，
       tenant/supplier/order 按实体去重，时间外推切分；切片=金额/租户/冲突/注入/PII/恢复
       train/dev/test=60/20/20，另有隐藏 challenge，不进入 Prompt 调试。

基线包括：确定性 Workflow（规则采购）、当前生产版本 model-route-v5/prompt-v8/orchestration-v3，以及简单模型路由。候选和基线固定工具/知识快照，各用 5 个随机种子配对运行，报告均值、方差和失败分类。禁止把同一合同改写后分到不同集合，也禁止用 challenge 结果反复调 Prompt，避免污染。

关键得分证据

目标契约、数据卡、污染检查、切片覆盖、可重复命令和基线结果。

达标与优秀

达标：来源、版本、切分和基线完整。优秀：隐藏集、方差、数据治理和污染响应。

可接受变体

小样本可用人工高质量金标，但需给置信范围和扩充计划。

常见扣分

只有 Prompt 列表；单一平均分；测试样本进入调参。

补学与复测

为新任务补数据卡、切片、污染控制和两个可复现基线。

B · 结果与轨迹

完整示范答案

Outcome 判断业务结果与约束，Trajectory 判断决策、工具、证据、授权、成本和恢复路径。定义允许多条合法路径和 hard fail；Trace 以版本、State diff、Decision、Observation、policy/effect refs 找首个偏离点，再映射正确 owner。

评价字段/判定
Outcomepurchase_id、权威金额/供应商、approval_valid、email_recipient、user_goal_met、hard_fail
Trajectoryrun/step/state_version+diff、decision/capability/args_hash、evidence IDs、policy decision、tool receipt、retry、latency/token/cost
允许路径先查合同再比价，或并行查材料后统一重排都可；只要证据、授权、预算和验收不变量满足
Hard fail任何跨租户 chunk、审批摘要失配仍执行、effect_unknown 盲重试均单独失败，不被最终文本分数抵消

示例失败：step2 的 RAG 候选包含 tenant B，policy decision 却是 allow；step3 模型据此选择低价供应商，step4 工具创建。首个偏离点是检索/治理未阻断，传播到推理和工具；不能把责任全部归模型。用相同 Trace 将候选替换为正确租户的 oracle context 后结果正常，可进一步支持归因；若仍错误才检查推理。

关键得分证据

outcome/trajectory rubric、可重放 Trace、首偏离归因和反事实替换。

达标与优秀

达标：结果、路径、hard fail、owner 完整。优秀：多合法路径、反事实和重放边界。

可接受变体

低风险纯文本任务可侧重 outcome，但仍需最低限度的证据和成本轨迹。

常见扣分

最终文本好就忽略越权路径；所有失败归模型；轨迹越短越好。

补学与复测

对一个失败 Run 标注首个偏离点、owner 与一条反事实验证。

C · 规则、人工与 LLM-as-a-Judge

完整示范答案

确定性格式、权限和 hard fail 先用规则；主观质量由带 rubric 的人工金标锚定；Judge 用于规模化近似，需换位/顺序控制、版本固定、与人工校准、分歧升级和注入防护。严重错误召回优先于平均相关性。

评分路由实例
确定性规则JSON Schema、citation 可定位、tenant 一致、金额阈值审批、receipt 幂等；任一 hard fail 直接 0
人工金标四维 rubric：结论正确性、证据充分性、取舍解释、可执行性；每维 1–4 有行为锚点，双人复标分歧仲裁
LLM Judge固定 judge-model/revision/prompt/rubric，输入脱敏且文档明确为数据；A/B 换位复判，低置信或高风险分歧升级人工

在 200 个金标样本上校准：报告 weighted kappa、与人工 Spearman、严重错误 recall 和各切片混淆矩阵。若 Judge 偏好冗长，加入“内容等价但长度不同”的成对样本并要求证据锚点；若换位结果翻转则不自动采信。Judge 不是事实来源，更不能覆盖规则检测到的越权；版本更新必须重新校准并监测线上抽样漂移。

关键得分证据

rubric、金标、一致性、混淆矩阵、严重错误指标、分歧样本和漂移监测。

达标与优秀

达标：三类评分器路由和校准清楚。优秀：置信区间、仲裁、注入红队和版本漂移。

可接受变体

双 Judge 可用于探索分歧，但不能用同源多数票替代人工标准或安全规则。

常见扣分

Judge 当真值；无人工锚点；把待评输出中的指令交给 Judge 执行。

补学与复测

建立 30 条金标，测 Judge 一致性、严重错误召回和分歧升级。

D · 回归与上线门禁

完整示范答案

候选与基线使用同一数据快照、依赖、Prompt/模型/工具版本和随机策略做配对重复实验；报告效应大小、方差/置信区间和关键切片。安全 hard fail 零容忍，质量做非劣/提升门槛，成本延迟设预算；通过后灰度，以线上 SLO 自动回滚。

配对结果（n=1000×5）：任务成功率 baseline 91.2%，candidate 93.0%，Δ=+1.8pp，95% CI [+0.7,+2.9]
注入切片：98.5%→99.4%；高金额切片：92.0%→91.8%，非劣界 -1pp（通过）
hard fail：baseline 1，candidate 0；p95 16.2s→17.4s（预算≤18s）
单位成功成本 ¥0.39→¥0.43（预算≤¥0.45）

机器门禁顺序：①跨租户/越权/重复副作用 hard fail 必须为 0；②所有关键切片满足非劣界且安全切片提升；③总体成功率置信区间下界>0；④成本与 p95 不超预算。通过后按 1%→5%→20% 灰度，每阶段至少覆盖规定流量和高风险样本；线上 hard fail>0、成功率下降>2pp、p95>20s 连续 10 分钟或接管率>12% 即自动回退旧 route/prompt/orchestration revision。

关键得分证据

配对报告、机器可读门禁、关键切片、灰度计划和回滚契约。

达标与优秀

达标：固定版本、重复、切片、非劣和灰度。优秀：统计功效、效应大小、自动回滚和失败审计。

可接受变体

低流量系统可延长灰度窗口或用影子流量，但仍要预定义停止条件。

常见扣分

一次平均分提升就全量；安全问题被平均；回滚只写人工观察。

补学与复测

为一项模型升级写配对实验、三层门禁和自动回滚条件。

E · 安全红队与线上反馈

完整示范答案

从资产、攻击者、入口、信任边界和影响建立威胁矩阵，在隔离租户用假凭据/canary 做多步注入、越权、泄露、投毒、工具滥用和资源耗尽。线上治理实时遏制；评测离线接收同意、脱敏、去重的 Trace/近失/人工接管，归因后加入版本化回归，修复经门禁再灰度。

红队路径隔离测试与预期
合同内间接注入→外发假合同要求发送通讯录；模型可引用文本但动作门禁必须 deny，canary 不离开沙箱
跨租户检索tenant A 查询 B 的唯一 canary；召回和 citation 均为 0
审批后参数漂移批准 60k 后改 80k；摘要重验拒绝，旧 receipt 不可复用
effect_unknown 重试下游成功后注入 timeout；系统先查 receipt，不创建第二单
资源耗尽循环搜索/长文档；预算和 sandbox quota 终止并产生类型化事件

线上治理仍实时阻断，不能在生产主动做破坏性攻击。经同意的 Trace 先删 PII、按事件/实体去重并做偏差校正；把近失事件和人工接管按首个偏离点归到编排、RAG、治理、工具或网关 owner，形成 security-regression-v7。修复必须重放原攻击与邻近变体，通过离线门禁后灰度；恢复证据包括策略/组件版本、受影响范围、零 canary 外发和 kill switch 演练。

关键得分证据

隔离红队计划、停止条件、最小复现、采样偏差说明、owner 和修复复测。

达标与优秀

达标：多步攻击、脱敏反馈、归因闭环。优秀：canary、近失、偏差校正和受控恢复。

可接受变体

自动攻击生成可扩展覆盖，但高风险样本和成功判据需专家审阅。

常见扣分

只测单轮越狱；生产做破坏性测试；盲目回灌线上 PII。

补学与复测

新增一个组合攻击，完成发现—归因—修复—回归—灰度证据链。
