校准纪律 · 接入层建立可信输入、呈现和恢复，不越权规划或执行业务动作。

A · 渠道、会话与身份

建议作答

connection 是传输连接，conversation 是用户可见对话，session 是连续交互上下文，subject 是服务端确认的授权主体，run 是一次目标执行。接入层规范化消息、locale、附件引用、客户端事件 ID 和 schema version；身份来自服务端链路，匿名转登录、群聊代理和跨设备都需显式重绑。

关键得分证据

五类 ID owner 图、去重键、身份上下文和重复/乱序事件 Trace。

达标与优秀

达标：五类边界、身份和规范化正确。优秀：匿名迁移、代理人、跨设备和版本兼容。

可接受变体

conversation/session 可在简单产品合并，但 subject 与 run 仍不能由客户端自报。

常见扣分

session 代表一切；信任 user_id；接入层直接执行退款。

补学与复测

重放匿名登录后的重复消息，证明身份和 Run 没有错误合并。

B · 流式与多模态

建议作答

流式协议包含 event_id、sequence、cursor、run/turn、类型、时间和 terminal status，区分 token、结构化进展、可撤回草稿和业务终态；断线用 cursor 重放或 snapshot，取消向下游传播并处理迟到事件。附件先上传、扫描、类型验证、ACL、OCR/转码和 provenance，再以 artifact 引用进入编排。

关键得分证据

背压、重放、取消、部分输出撤回、低置信 OCR 和删除传播 Trace。

达标与优秀

达标：事件协议与附件治理完整。优秀：背压、snapshot、跨设备恢复和低置信降级。

可接受变体

SSE、WebSocket 或轮询均可，只要顺序、恢复、取消和终态语义明确。

常见扣分

只流 token；HTTP 200=业务完成；附件原样直送模型。

补学与复测

模拟断线、取消和 OCR 低置信，验证 cursor、撤回与安全降级。

C · 澄清与 HITL

建议作答

澄清只询问阻塞决策的最小信息，记录 question_id、缺口、可选值和到期；HITL 暂停持久化 checkpoint。审批展示人可理解的动作摘要，并把 decision receipt 绑定 subject、action、resource、参数摘要、策略版本、有效期和 checkpoint；恢复前重验，参数漂移则失效。

关键得分证据

澄清/审批/恢复时序、迟到答复、取消、重复决定和可访问呈现。

达标与优秀

达标：问题—决定—checkpoint 关联和摘要绑定。优秀：并发回答、过期、参数漂移和多渠道等价。

可接受变体

低风险缺省值可自动采用，但必须向用户可见、可撤销且不扩大权限。

常见扣分

循环追问；“确认吗”无动作摘要；按钮点击直调工具。

补学与复测

用审批后金额变化场景验证旧 receipt 失效和安全恢复。

D · 人工移交

建议作答

handoff package 只包含原因、目标、已确认事实与来源、推断、未决项、已执行 receipt、effect_unknown、权限范围、checkpoint/resume token 和队列 SLA。状态机支持 requested/queued/accepted/in-human/resolved/returned/cancelled；人工结果结构化回写，Agent 恢复前处理版本和并发冲突。

关键得分证据

最小权限包、人工不可用降级、双向接力、超时和未知副作用记录。

达标与优秀

达标：最小 handoff 与双向状态完整。优秀：并发控制、SLA 分级、回交和审计。

可接受变体

人工可直接结束 Run 或交回 Agent；两种路径都需业务终态和证据。

常见扣分

转人工即丢状态；复制全部 transcript；人工修改后 Agent 用旧状态继续。

补学与复测

演练人工队列不可用和人工交回 Agent 两条路径。

E · 同意、恢复与可访问性

建议作答

Consent receipt 记录目的、数据/动作范围、主体、版本、时间、有效期和撤回方式，撤回沿缓存与派生传播。错误按 validation/policy/business/transient/partial/effect_unknown 映射为重试、澄清、查询、转人工或停止。可访问性覆盖键盘、焦点、读屏标签、状态播报、非颜色提示、触控目标和多模态等价。

关键得分证据

consent receipt、错误矩阵、撤回传播、跨设备恢复和辅助技术完成率。

达标与优秀

达标：同意、类型化恢复和 WCAG 核心路径完整。优秀：partial/effect_unknown、跨设备、真实辅助技术和可追溯验收。

可接受变体

某些必要处理依据合同/法规而非 consent，但仍需告知、最小化和可审计依据。

常见扣分

consent=true；所有错误都重试；只测颜色对比；流式更新不播报状态。

补学与复测

用键盘和读屏完成一次 effect_unknown 恢复，并验证撤回传播。
