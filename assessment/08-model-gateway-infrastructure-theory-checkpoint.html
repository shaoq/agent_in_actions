统一场景

你要为多租户企业采购 Agent 建设模型网关和运行平台。系统在华东、华南部署，支持合同长文审查、低延迟分类、结构化工具调用和用户 Python 分析；高金额动作需审批。主模型偶发 429 和流式中断，峰值 20 RPS、平均服务时间 8 秒、突发系数 1.5；租户数据不得跨域或跨租户，Python 只能读取本次授权文件且不能访问公网和云 metadata。请设计可上线的平台方案。

证据要求

  1. 一份模型网关请求、流式事件、响应与错误 schema，以及 capability catalog、route policy version 和两次可解释路由记录。
  2. 一张覆盖未派发拥塞、429、首 token 超时、部分流式输出、区域故障与预算耗尽的处理矩阵；说明哪些可重试、降级、排队或 fail closed。
  3. 一份容量计算：峰值并发基线为 20 × 8 × 1.5 = 240，再说明 headroom、RPM/TPM、输出长度分布、队列和过载拒绝；提供 token/价格/延迟归因字段。
  4. 一张运行拓扑与状态表：run checkpoint、队列、artifact、cache、trace/audit 的 owner、持久性和恢复保证；模型 egress 与工具 egress 必须分离。
  5. 一次沙箱设计评审：execution_id 绑定 tenant/run/image/policy/mount/network/secret/quota/deadline，并给出跨租户、SSRF/metadata、路径穿越、secret 泄露、资源耗尽和销毁残留测试结果。
