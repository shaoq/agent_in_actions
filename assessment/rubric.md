# Agent 能力考核标准

## 评分维度

### 1. 架构理解

- 1 分: 只能描述 LLM 调用流程。
- 3 分: 能说出 Agent 的主要组件。
- 5 分: 能清楚说明组件边界、依赖关系和常见错误设计。

### 2. 工程实现

- 1 分: 只能跑通简单 demo。
- 3 分: 能实现工具调用、RAG 或记忆中的单项能力。
- 5 分: 能实现带失败处理、权限、日志和评测的端到端 Agent。

### 3. 调试能力

- 1 分: 主要靠改 prompt 试错。
- 3 分: 能根据日志定位大致问题。
- 5 分: 能基于 trace 区分模型、检索、工具、状态、权限、评测问题。

### 4. 安全与治理

- 1 分: 只知道 prompt injection 这个概念。
- 3 分: 能设计基础权限和审计。
- 5 分: 能覆盖在线防护、离线安全评测和沙箱隔离。

### 5. 面试表达

- 1 分: 回答零散,缺少结构。
- 3 分: 能按模块回答常见问题。
- 5 分: 能结合项目经验、取舍、失败案例和上线策略回答追问。

## 阶段考核任务

1. 设计一个客服 Agent,要求覆盖 RAG、工具调用、权限、安全、评测。
2. 给定一段失败 trace,判断问题来源并给出修复方案。
3. 实现一个最小可运行 Agent demo,包含至少一个工具和一个评测用例。
4. 解释 Memory 与 RAG 的边界,并设计防污染策略。
5. 设计一套 Agent 上线前评测与上线后观测方案。
