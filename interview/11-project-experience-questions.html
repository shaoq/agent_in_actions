PRJ-01基础

三分钟介绍一个 Agent 项目

考察目标：在有限时间讲清问题、角色、方案、结果和反思。

对应能力：九层架构表达

主问题：请选择你最熟悉的 Agent 项目，说明业务问题和个人贡献。

连续追问：

  1. 你亲自负责的边界是什么？
  2. 上线前后哪个指标发生了变化？
  3. 如果没有 LLM，最强基线是什么？

练习要求：使用真实项目，录制 3 分钟版本和 30 秒摘要。

PRJ-02进阶

为什么选择 Agent 而不是普通工作流？

考察目标：证明架构选择来自任务不确定性而非追逐技术。

对应能力：编排模式取舍

主问题：你的项目为什么需要动态决策或工具使用？

连续追问：

  1. 哪些阶段仍是确定性 workflow？
  2. 最简单替代方案表现如何？
  3. Agent 带来的额外复杂度是什么？

练习要求：提供基线、约束和实际对比，不使用“更智能”作为结论。

PRJ-03进阶

最难的技术取舍是什么？

考察目标：观察候选人是否能用约束和数据做决策。

对应能力：Agent 学习首页

主问题：描述一次你在质量、延迟、成本或安全之间做的真实取舍。

连续追问：

  1. 当时有哪些备选方案？
  2. 决定依赖了哪些测量数据？
  3. 什么条件变化会让你改选另一方案？

练习要求：给出约束、方案矩阵、决定、结果和反事实。

PRJ-04综合

讲一次失败或线上事故

考察目标：观察归因、止损、责任意识和系统改进。

对应能力：治理与事件响应

主问题：描述一次 Agent 结果错误、工具失败或质量回归，你做了什么？

连续追问：

  1. 最早的告警或用户信号是什么？
  2. 根因证据如何排除其他假设？
  3. 修复后新增了哪些门禁？

练习要求：只讲亲历事件，覆盖影响、时间线、根因、修复和预防。

PRJ-05综合

你如何定义和提升项目效果？

考察目标：区分业务结果、模型指标和过程指标。

对应能力：评测层总览

主问题：项目上线前后使用了哪些离线和在线指标？

连续追问：

  1. 数据集从哪里来，是否有偏差？
  2. 指标提升是否显著且可归因？
  3. 质量与成本冲突时如何决策？

练习要求：填入真实基线、样本量、切片、变化和决策门槛。

PRJ-06进阶

如何优化延迟和成本？

考察目标：观察是否先测量再优化，并保持质量底线。

对应能力：成本、延迟与容量

主问题：描述一次实际的 token、路由、缓存或流程优化。

连续追问：

  1. 瓶颈在首 token、模型生成还是工具？
  2. 优化对关键质量切片有什么影响？
  3. 如果流量增长十倍，方案还成立吗？

练习要求：提供优化前后 P50/P95、单位任务成本和质量回归证据。

PRJ-07进阶

如何与产品、安全和业务协作上线？

考察目标：理解 Agent 上线是跨角色风险决策。

对应能力：学习与上线能力

主问题：描述一次你推动需求澄清、风险评审和灰度发布的过程。

连续追问：

  1. 哪些需求被你拒绝或降级？
  2. 安全和业务冲突如何解决？
  3. 上线后谁拥有告警和回滚权？

练习要求：说明角色、决策记录、门禁、灰度、owner 和沟通结果。

PRJ-08综合

如果重做一次，你会改变什么？

考察目标：观察复盘深度和对复杂度的重新判断。

对应能力：能力证据与复测

主问题：基于当前证据，你会保留、删除或重构哪些部分？

连续追问：

  1. 当时为什么没有这么做？
  2. 哪个假设后来被证明错误？
  3. 下一版怎样用更小实验验证？

练习要求：给出历史约束、失效假设、新方案和验证计划，避免事后诸葛亮。
