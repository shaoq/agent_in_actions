A. 系统设计

SYS-01综合

设计电商退款 Agent

考察目标：覆盖身份、订单证据、工具副作用、审批和 effect unknown。

对应能力：九层架构

主问题：设计支持查单、退款、改址和人工争议处理的生产系统。

连续追问：

  1. 高金额退款如何暂停和恢复？
  2. 退款接口超时但可能成功怎么办？
  3. 上线前用什么数据集和门禁？

练习要求：20 分钟内给出 SLO、主数据流、权限、幂等、trace 和评测。

B. Trace 调试

统一作答格式：症状 → 已知证据 → 分层假设 → 最小验证 → 修复 → 回归。没有 trace 支持的结论只能称为假设。
DBG-01综合

Agent 重复调用同一工具

考察目标：区分模型、observation、状态写回与终止条件故障。

对应能力：编排总览

主问题：trace 显示连续 5 步调用 search，参数相同，结果也相同，如何定位？

连续追问：

  1. 先检查哪些 state diff 和 event？
  2. 如何证明是 observation 未写回而非模型能力差？
  3. 修复后用什么循环检测回归？

练习要求：提出至少三个可证伪假设并按成本排序验证。

  3. 修复 chunk 或 rerank 后怎样做切片回归？

练习要求：列出 document version、retrieved IDs、scores、context 和 claims 证据。
