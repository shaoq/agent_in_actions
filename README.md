# Agent in Actions

面向 AI Agent 应用工程师的学习、实战与考核工程。

这个项目的目标是把 Agent 相关知识从零散概念整理成一套可训练、可复盘、可面试表达、可工程落地的能力体系。它不是资料仓库,也不是单纯的 demo 集合,而是围绕“成为能通过 Agent 应用工程师面试并胜任真实研发”的长期训练系统。

当前阶段:第一阶段“学习与训练”中的 L0 Agent 基础。采用理论优先路线,理论门禁通过后再进入实验。

当前学习单元:Agent 基础语义。

## 项目使命

本项目服务于一个明确目标:系统掌握 Agent 应用工程的核心技术,最终能够在公司面试中清楚证明自己的架构理解、工程实现、调试分析、安全治理和评测能力。

所有新增内容都必须回答至少一个问题:

1. 它是否帮助我更理解 Agent 系统如何工作?
2. 它是否帮助我把 Agent 能力落成可运行工程?
3. 它是否帮助我在面试中更清楚地表达取舍?
4. 它是否帮助我发现自己哪里还不会?

如果答案都是否,就不应该放进这个项目。

- `interview/`: 面试题、系统设计题、追问链和表达模板。

## 当前重点

1. 完成 Agent 基础理论,建立 Agent、Workflow、状态、动作、环境和反馈的共同语义。
2. 通过 Agent 基础无代码检查点,再复述九层 Agent 架构。
3. 按五单元学习编排理论,通过编排理论门禁后再运行最小 Agent Loop。
4. 验证理论优先方法后,逐个补齐推理、工具、RAG、记忆、治理、评测和基础设施理论。

## 学习闭环

每个主题都按理论优先闭环推进:

1. 知识输入:读文档、论文、官方说明或源码。
2. 架构抽象:整理这个主题在 Agent 系统中的位置、职责和边界。
3. 理论推演:完成概念图、模式比较、失败分析和连续追问。
4. 理论门禁:不用代码完成边界解释和系统设计。
5. 工程实验:门禁通过后用最小 demo 验证理论。
6. 面试与考核:整理表达并用新案例复测。

不建议只阅读不产出。每次学习至少留下一个可复用资产:一页文档、一段实验代码、一组问答、一个复盘记录。

## 建议学习顺序

1. Agent 基础语义与九层架构全景。
2. 编排控制层与推理内核。
3. 工具 / 动作层与 MCP。
4. 知识 / RAG 和记忆系统。
5. 治理与可观测、评测层。
6. 模型网关、基础设施与交互接入。
7. 单 Agent 理论稳定后再学习 Multi-Agent。

## 产出标准

新增专题文档应包含:

- 这个主题解决什么问题。
- 它在 9 层 Agent 架构中的位置。
- 核心概念和关键术语。
- 生产环境中的设计取舍。
- 常见失败模式。
- 面试高频追问。
- 后续实验入口。

新增实验应包含:

- 运行方式。
- 实验目标。
- 关键代码路径。
- 预期现象。
- 失败场景。
- 可扩展方向。

新增面试材料应包含:

- 问题。
- 短回答。
- 深入回答。
- 面试官可能追问。
- 工程案例表达。

## 防跑偏规则

- 不把本项目变成资料下载目录。
- 不把本项目变成纯前端展示项目。
- 不长期停留在概念总结,必须逐步补实验。
- 不为了追热点新增主题,除非能映射到架构、实验或面试能力。
- 不把无关业务文件、Excel、临时脚本放进根目录。
- 不把“模型 API 调用”误当成完整 Agent 工程能力。
