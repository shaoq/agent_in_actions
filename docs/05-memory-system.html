UNIT.02

架构位置与边界

职责边界 · 记忆记录主体相关的经历、偏好和已验证经验；RAG 检索外部资料；Run State 控制一次执行。存储介质相同不代表语义相同，生命周期、权限与删除策略必须按语义归属。
契约面组件级定义
写入输入事件、主体、来源、同意、价值、敏感度和保留期
读取输出候选记忆、来源、时间、置信、适用范围和冲突信息
上游编排/交互产生经历，治理执行同意与隐私策略
下游推理使用候选记忆，但不得把历史偏好当事实指令

UNIT.03

模式与取舍

MODE 1

工作记忆

单次决策窗口；快速但随运行结束。

MODE 2

情景记忆

具体经历与结果；可追溯但易积累噪声。

MODE 3

语义记忆

从多次经历整合的稳定偏好；紧凑但需证据门槛。

MODE 4

程序记忆

已验证流程与策略；复用强但版本和权限风险高。

UNIT.05

生产工程约束

  • 默认不写入，先评估价值、同意、敏感度和保留期。
  • 每条记忆保留来源、主体、时间、版本和置信。
  • 检索结果视为候选上下文，需与当前指令和证据冲突检查。
  • 支持查看、更正、导出、遗忘和删除传播。
UNIT.06

面试表达

30 秒回答

记忆系统保存的是 Agent 经历，而不是外部知识。它通过写入门禁、主体隔离、检索、整合、衰减和遗忘，在未来任务中提供有来源的候选经验，同时必须支持隐私控制、纠错和删除。

3 分钟回答结构

先区分 state、session、memory 和 RAG，再讲四类记忆及生命周期；用写入价值、主体范围、时效冲突和投毒防护说明治理；最后用命中率之外的任务增益、误用率和删除可验证性评估。

表达检查 · 不要只报框架或产品名；必须讲清输入输出、owner、证据、失败归因和为什么选择该模式。
