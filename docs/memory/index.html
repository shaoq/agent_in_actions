记忆系统理论课程

记忆系统（Memory System）负责按策略沉淀和回忆 Agent 与用户的经历：偏好、历史事件、已确认事实、成功程序和任务轨迹。它要回答“该不该记、记什么、何时取、何时改、何时忘”，并保留来源、授权、置信度和生命周期证据。

学习目标

  1. 区分 run state、session context 与跨会话长期记忆。
  2. 按工作、情景、语义、程序记忆选择数据结构与使用场景。
  3. 设计候选写入、验证、整合、读取、衰减、删除的完整生命周期。
  4. 处理隐私、跨租户泄露、间接注入、记忆投毒和被遗忘权。
  5. 用 trace 区分“没记、记错、取错、整合错、忘错”和推理失败。

前置知识与学习顺序

前置为九层架构、编排控制、推理内核和知识 / RAG。推荐顺序：

顺序理论单元完成产出
01状态、会话与记忆边界三层生命周期与 owner 表
02工作、情景、语义与程序记忆类型选型与来源契约
03读取、写入、整合与遗忘读写门禁和删除闭环
04隐私、投毒与治理威胁模型与治理控制

组件边界

记忆不是知识 / RAG · 记忆系统沉淀用户与 Agent 的经历；知识 / RAG 检索产品手册、制度、代码文档等外部资料。两者可以使用相似的存储或检索技术，但来源、写入权、更新频率、共享范围和删除义务完全不同。
相邻组件它负责记忆系统不负责
交互与接入身份、会话通道、用户同意入口不把 session_id 当长期记忆策略
编排控制run 状态、何时读写、失败恢复不拥有工作流推进和终止判定
推理内核基于候选记忆做决策不把模型生成内容直接认证为事实
知识 / RAG检索外部、可追溯资料不把公共文档复制为个人经历
治理与可观测策略、审计、脱敏、告警不绕过统一身份与授权

完成标准

  • 能以同一案例划清 run state、session context、长期记忆与外部知识。
  • 能设计带 provenance、scope、confidence、retention 和 consent 的记忆契约。
  • 能解释读取排序、写入门禁、冲突整合、衰减与物理删除。
  • 完成记忆系统理论检查点：总分至少 80，且五项任务各至少 12/20。

available 仅表示学习资产可用；个人掌握状态以门禁证据与复测记录为准。
