推理内核理论课程

推理内核（Reasoning Core）是 Agent 的决策面：它读取由编排层准备的决策上下文，识别目标、约束、证据与缺口，选择下一步，并返回可被程序验证的结构化意图。它不是整个 Agent，也不拥有任务生命周期、工具执行或模型厂商连接。

学习目标

完成本课程后，你应能：

  1. 把自然语言目标转成稳定的决策输入、输出与不变量，而不是只讨论 prompt。
  2. 设计判别联合（discriminated union）形式的 Decision schema，区分 act、respond、clarify、escalate 与 finish。
  3. 根据任务可枚举性、观察依赖、风险和预算选择 Direct、Routing、Plan-and-Execute、有限 ReAct 或自检模式。
  4. 把不确定性显式化，并在补充信息、检索、修复、降级和转人工之间选择安全出口。
  5. 依据 trace 区分推理失败、编排失败、工具失败与模型网关失败，提出可验证的修复。

前置知识

  1. 通过L0 Agent 基础理论检查点。
  2. 完成L1 九层 Agent 架构并通过架构检查点。
  3. 完成编排控制层理论课程。本课程默认你已理解 run、state、step、observation 和终止条件。
学习顺序 · 编排先于推理，是因为必须先知道“谁驱动循环、谁拥有状态”，才能把模型决策放进正确边界。

四个理论单元

顺序单元核心问题学习证据
1角色、职责与决策契约推理内核收到什么、保证什么、明确不做什么？边界表 + Decision contract
2结构化输出与工具选择如何让模型建议变成可校验的动作意图？schema + 校验与修复流程
3推理模式及取舍何时一步判断，何时规划、观察或自检？模式选择矩阵
4不确定性、失败与恢复不知道、做不到或证据不足时如何安全退出？失败归因表 + 恢复策略

组件边界

组件拥有的职责与推理内核的接口不得混入推理内核
编排控制层run 生命周期、状态推进、预算、重试、终止与恢复传入 DecisionContext；接收 Decision 后校验并推进while loop、checkpoint、任务终态写入
推理内核理解当前问题、比较候选、选择意图、表达不确定性输入上下文；输出结构化候选决策及证据引用直接执行副作用、越权改变生命周期
治理与可观测层策略、权限、审计、trace、敏感信息与人在回路向决策上下文注入约束；验证高风险意图把“模型觉得安全”当成授权
工具 / 动作层工具 schema、调用、超时、幂等、结果与类型化错误推理只选择能力并提出参数；工具层产生 observation在推理代码里直接发邮件、退款或写数据库
模型网关供应商适配、模型路由、fallback、配额、缓存与调用观测推理提交能力需求，不绑定厂商 SDK推理内核直连任一模型厂商

这些组件可以部署在同一进程，但语义必须分开。进程边界决定部署，职责边界决定谁能改变什么状态、谁应为失败负责。

学习方法

  1. 第一次阅读：只画输入、输出、不变量和非职责。
  2. 第二次阅读：为“企业工单助手”写一份 Decision schema，并故意构造三个非法输出。
  3. 场景迁移：分别为客服分类、开放调研和高风险退款选择模式，写出拒绝替代方案的理由。
  4. 失败复盘：严格分开 trace 事实、原因假设、补充证据、修复层和回归用例。
  5. 口述训练：用 30 秒定义、3 分钟方案、两轮追问表达同一套边界。

完成标准

“页面可用”不等于“个人掌握”。课程资产标为 available 只说明你现在可以学习；完成需同时满足：

  • 读完四个单元，并提交每个单元要求的学习证据。
  • 能脱稿解释推理内核与编排、工具、治理、模型网关的边界。
  • 完成推理内核理论检查点，总分至少 80，且每项至少达到该项 60%。
  • 把首次结果、缺口、补学动作和复测结果登记到进度跟踪。

课程范围

  • 本课程讨论工具选择的决策语义，不展开 MCP、工具注册、权限执行、重试与幂等实现。
  • 讨论模型能力需求与失败，不展开模型路由、缓存和厂商 fallback 实现。
  • 讨论推理 trace 的必要证据，不记录或要求暴露模型私有 chain-of-thought。
  • 不包含 L3 可运行实验和 L4 独立面试题库。
