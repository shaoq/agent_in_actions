UNIT.02

架构位置与边界

职责边界 · 治理在实时请求上允许、拒绝、审批或降级；可观测记录事实。离线评测消费脱敏 trace 学习质量规律；基础设施沙箱执行隔离。安全不是一个独立万能层，而是三处协同。

UNIT.03

模式与取舍

MODE 1

策略即代码

规则稳定、需审计；确定性强但维护成本高。

MODE 2

风险分级

按动作和数据分类；体验与安全平衡，但需持续校准。

MODE 3

HITL 门禁

不可逆或高影响动作；安全但会增加等待与运营成本。

MODE 4

全链路 Trace

跨模型、工具与检索；诊断强但需采样和脱敏。

UNIT.05

生产工程约束

  • 控制决定和策略版本必须写入不可抵赖审计。
  • 敏感内容最小采集，日志与 trace 分级访问。
  • 预算覆盖 token、步骤、时间、并发和副作用次数。
  • 事件响应包含检测、限制、恢复、复盘和回归验证。
UNIT.06

面试表达

30 秒回答

治理与可观测是在线横切控制面：治理决定请求和动作能否继续，可观测留下跨组件证据。它不替代离线评测，也不替代基础设施沙箱；三者分别负责实时控制、离线学习和执行隔离。

3 分钟回答结构

从身份和策略决策链开始，说明 trace/log/metric 的不同用途与关联 ID；再讲注入、PII、HITL 和预算；最后说明告警、事件响应如何产生脱敏样本并反馈评测与编排。

表达检查 · 不要只报框架或产品名；必须讲清输入输出、owner、证据、失败归因和为什么选择该模式。
