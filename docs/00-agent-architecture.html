组件构成

1
交互与接入层
INTERACTION / CHANNEL

agent 与外界的边界。把不同渠道(网页、语音、IM、IDE)的输入统一成内部表示,并管理"一次会话"的生命周期。

职责:渠道适配、会话状态持有、人在回路入口。它不思考,只翻译和持会话。

边界 · 不含推理;多渠道协议差异在此收敛。

[ 待补充 ] 内部构成 · 子组件 · 渠道适配模式 · 会话生命周期 · 设计权衡
2
编排控制层 ★
ORCHESTRATOR · 系统大脑

整个系统唯一驱动者。决定"下一步做什么":再想一轮、调工具、查记忆、还是拆任务给子 agent。

职责:单 agent 循环、多 agent 协作(orchestrator-workers / 层级 / 黑板)、状态机、规划-反思循环。所有其他组件都由它驱动。

边界 · 本月研究里最被强调的一层;架构分水岭。

[ 待补充 ] 单 vs 多 agent 决策 · 协作模式对比 · 状态机选型 · 规划-反思循环
3
推理内核
REASONING CORE

agent 的"思考引擎":LLM + 推理模式(ReAct、Plan-and-Execute、Reflexion 等)。

职责:给定上下文,产出下一步判断。本身无状态、不记事、不直接动手。

边界 · 永不直连模型厂商,必经 ⑨ 模型网关。
深入展开(待补充)
[ 待补充 ] 推理模式对比 · 自适应思考/effort · 模型选型分流
4
工具 / 动作层
TOOLS / ACTIONS

agent 的"手和脚"。"能聊天→能动手"转变的落点:MCP 工具、浏览器操作、computer-use、可复用 skills。

职责:声明能力 + 调用协议。动作的真正执行下沉到 ⑨ 的沙箱。

边界 · 声明在此,执行在沙箱;不裸跑生产环境。

[ 待补充 ] MCP 协议 · browser/computer-use 子系统 · skills 封装 · action 工具占比趋势
5
记忆系统 ★
MEMORY · 分水岭

agent 自己产生的状态历史,跨会话复用。分三类:情景(发生过什么)、语义(持久事实/业务上下文)、working(当前任务草稿)。

职责:把记忆当设计时一等公民,而非事后补丁。

边界 · 记忆是 agent"经历"出来的;和 ⑥ 知识(外部已有)严格分离。

6
知识 / 检索
KNOWLEDGE / RAG

外部既有的信息库:文档、向量库、API 数据。RAG、检索增强归这里。当 agent 自己记忆不够时,来这里查。

职责:提供事实依据,降低幻觉。

边界 · 只读为主;不承载 agent 的对话历史(那是 ⑤)。
深入展开(待补充)
[ 待补充 ] 检索策略 · 向量库选型 · chunking · 混合检索 · 与记忆协作
7
治理与可观测层 ⇆
GOVERNANCE · 横切

在线横切的防护与监督。三组职责:

安全防护:prompt injection 防御 · 输入/输出护栏 · 数据安全(PII/脱敏) · 内容安全。

访问控制:权限 · 授权 · 审计(进出都校验、可追溯)。

运行监督:trace · 失败监测 · 成本预算/限流。每次实时请求都过它,不参与推理,但是上线前置条件。离线安全评测见 ⑧,执行隔离见 ⑨。

边界 · 只管"在线·每次请求";离线(安全)评测归 ⑧,执行隔离(沙箱)归 ⑨,成本策略归 ⑨。
深入展开(待补充)
[ 待补充 ] prompt injection 防御 · 护栏蓝图 · 权限模型 · 审计 · 数据安全/PII · 人在回路
8
评测层 ⟂
EVALS · 离线旁路

回答"agent 干得好不好、有没有退化、危不危险"。两类评测:

功能:golden cases、自动评分(LLM-as-judge/规则)、回归测试、基准对标(Terminal-Bench)、版本 A/B。

安全:红队测试、能力扫描(MakerChecker 类)、越狱测试。离线/周期度量,不参与实时请求;消费 ⑦ 的 trace,结果反馈 ②。

边界 · 离线,不在请求路径;含安全评测;与 ⑦(在线防护)是上下游而非同层。

9
模型网关 + 基础设施
MODEL GATEWAY · INFRA

推理内核的"供水系统"。模型网关统一处理多模型路由、降本策略(开源/前沿分流)、缓存、故障转移;底下是沙箱执行环境和运行时存储。

职责:把模型能力从 ③ 解耦,这是控成本、可切供应商的关键。成本策略归此,成本预算/限流归 ⑦。

边界 · 所有模型调用必经此;执行隔离(沙箱)在此 - 防 agent 裸跑/能力溢出。

FIG.03

七条关键关系

R1 ② 是唯一驱动者:③④⑤⑥ 都是被编排层调用的"能力源",彼此不直接耦合,靠 ② 协调。
R2 ⑦ 在线横切一切:任何请求进、任何动作出,都强制穿过治理层,没有旁路。
R3 ③ 经 ⑨ 取模型:推理内核永远不直连模型厂商,必经网关,才能路由、降本、故障转移。
R4 ⑤ 与 ⑥ 分离:记忆是私历、知识是公库,混在一起是常见架构错误。
R5 ④ 的执行下沉 ⑨:动作声明在 ④,真实执行在沙箱,避免 agent 裸跑生产环境。
R6 ⑧ 评测是离线旁路:消费 ⑦ 的 trace 对行为打分,结果反馈 ② 优化;不进实时请求路径。
R7 安全是横切,三处分布:在线防护归 ⑦、离线安全评测归 ⑧、执行隔离(沙箱)归 ⑨;不单建安全层。
