知识 / RAG 理论课程

知识 / 检索层（Knowledge / Retrieval）负责从外部、可追溯资料中摄取、索引和检索证据，并把带来源、版本与访问范围的候选上下文交给编排和推理内核。RAG（Retrieval-Augmented Generation）是“先取证、再生成”的系统模式，不是单个向量数据库产品。

学习目标

  1. 解释离线摄取与在线检索两条流水线，以及每个阶段的输入、输出和证据。
  2. 围绕结构、语义、版本和权限设计 chunk 与索引。
  3. 区分 recall、rerank、context packing 与 generation 的目标和指标。
  4. 实现“检索前授权、生成后校验”的 grounding、引用与访问控制心智模型。
  5. 用分层评测和 trace 定位 coverage、index、retrieval、rerank、generation 与 citation 失败。

前置知识与顺序

前置为九层架构、编排控制、推理内核和工具 / 动作层。推荐依次完成以下单元：

顺序理论单元可交付理解
01边界与端到端流水线职责图、在线/离线链路和接口契约
02摄取、切分与索引可删除、可回滚、保留 ACL 的索引方案
03检索与重排召回漏斗、混合检索与无答案策略
04Grounding、引用与访问控制声明—证据绑定和权限不扩散设计
05评测与失败诊断分层指标、golden set 与 trace 归因

组件边界

知识不是记忆 · 知识 / RAG 检索外部资料，如产品手册、制度和代码文档；记忆系统沉淀 Agent 与用户的经历，如历史决策、偏好和任务轨迹。两者可由编排共同使用，但来源、写入、授权、保留期和污染治理必须独立。
相邻组件它负责知识 / RAG 不负责
编排控制决定何时检索、保存 step 状态、处理降级不拥有 run 生命周期或完成判定
推理内核基于授权证据形成回答或动作意图不替模型决定目标，也不保证生成忠实
记忆系统按策略读写 Agent 经历和用户偏好不把一次会话经历自动写入公共知识索引
治理与可观测身份策略、审计、敏感信息与 trace不绕过治理自行扩大资源范围
评测层离线消费 trace、运行回归与安全评测不把线上检索分数等同最终质量结论

完成标准

  • 完成五个单元的学习产出，能画出离线与在线链路并标出权限传播。
  • 能用 trace 判断“没资料、没索引、没召回、被重排、没 grounded、引用错”中的具体阶段。
  • 能以同一案例清楚区分外部知识和 Agent 经历记忆。
  • 完成知识 / RAG 理论检查点：总分至少 80，且五项任务各至少 12/20。

available 只表示资产可用；是否掌握仍以进度页中的证据和复测为准。
