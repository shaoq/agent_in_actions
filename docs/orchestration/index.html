编排控制层理论课程

定位

本课程属于 L2 组件理论深潜。目标是在阅读代码前,充分理解编排控制层的职责、状态、模式、可靠性和 Multi-Agent 边界。

前置条件:

  1. 阅读Agent 基础理论。
  2. 通过Agent 基础理论检查点。
  3. 完成九层 Agent 架构,并通过L1 九层架构检查点。

五个理论单元

顺序 单元 核心问题 必须产出
1 职责与边界 编排、推理、Workflow、状态机、工具和治理如何分工? 边界表、请求控制图
2 状态与生命周期 run、session、checkpoint 和 memory 如何区分与演进? 状态模型、生命周期图
3 模式与取舍 何时使用 Workflow、Router、ReAct 或 Plan-and-Execute? 模式选择矩阵
4 可靠性与恢复 如何停止、重试、恢复、补偿、转人工并留下证据? 失败矩阵、trace 归因
5 Multi-Agent 边界 什么时候分工收益超过协调成本? 升级决策表

必须按顺序学习。Multi-Agent 放在最后,因为不了解单 Agent 状态与可靠性时,很难判断多 Agent 的新增复杂度。

六维学习框架

每个单元都从以下维度展开:

  1. 定义与动机。
  2. 架构边界。
  3. 内部机制。
  4. 方案模式。
  5. 工程取舍。
  6. 失败与治理。

文档中的结论分为三类:

  • 通用事实:不依赖本项目的稳定机制或工程约束。
  • 项目参考架构:本项目用于学习和讨论的九层职责划分。
  • 实现偏好:根据风险、团队和技术栈可替换的具体选择。

学习方式

每个单元按以下步骤完成:

  1. 第一次阅读:只回答“为什么存在”和“边界是什么”。
  2. 第二次阅读:自己重画图表,不要复制原图。
  3. 场景推演:对给定业务选择模式并说明拒绝其他方案的理由。
  4. 失败分析:区分 trace 事实、原因假设和修复层次。
  5. 口述复盘:3 分钟主回答 + 至少两轮追问。

建议时长只用于安排节奏,每个单元可以分多次完成。是否结束由产出质量决定。

理论门禁

完成五个单元后,执行编排无代码理论检查点。通过前暂不要求运行实验。

门禁证据:

  • 单 Agent 请求链路图。
  • run/session/checkpoint/memory 边界说明。
  • 编排模式选择矩阵。
  • decide/act/observe/terminate 失败归因。
  • 可暂停工单 Agent 无代码系统设计。
  • 连续口述追问记录。

总分达到 80 且每个维度达到单项门槛后,再进入最小 Agent Loop 实验,用代码验证理论。

课程边界

  • 本课程解释 tool action 与编排的接口,不深入 MCP、schema、权限实现和幂等代码。
  • 本课程解释 checkpoint 与 memory 的语义差异,不深入长期记忆检索算法。
  • 本课程讨论 trace 证据,不实现可观测平台。
  • 本课程讨论 Multi-Agent 选择,不实现多 Agent 实验。
