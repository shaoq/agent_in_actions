UNIT.02

架构位置与边界

职责边界 · 评测主要离线消费结果与 trace，不参与每个在线请求的即时授权；在线治理负责实时控制。评测发现问题后反馈编排、工具、RAG、记忆、网关或交互 owner，而不是只换模型。
契约面组件级定义
输入版本化任务、期望、rubric、环境、结果、trajectory
输出指标、置信区间、失败分类、样例证据和门禁决定
上游线上脱敏 trace、人工反馈、红队发现与业务风险
下游发布系统、编排策略和各组件修复 backlog

UNIT.03

模式与取舍

MODE 1

规则评分

格式、约束、确定事实；稳定但覆盖有限。

MODE 2

人工评分

主观质量与高风险判断；可信但昂贵慢。

MODE 3

LLM Judge

规模化语义评估；灵活但有偏差和泄漏风险。

MODE 4

混合裁决

规则筛选、Judge 扩展、人工仲裁；生产更稳健。

UNIT.05

生产工程约束

  • 数据、prompt、模型、工具和环境版本必须可复现。
  • 门禁指标包含总体、关键切片和硬安全约束。
  • 评测样本与生产隐私策略一致，避免训练/测试泄漏。
  • 失败必须带证据并归因到可行动的组件 owner。
UNIT.06

面试表达

30 秒回答

评测层离线验证 Agent 的结果和轨迹，用版本化数据、规则、人工或 Judge 形成指标与上线门禁。它消费线上 trace 并反馈编排和各组件，但不替代在线治理的实时授权与阻断。

3 分钟回答结构

先从业务目标、风险和数据集代表性讲起，再区分 outcome 与 trajectory；解释规则、人工、Judge 的组合和校准；随后讲统计波动、回归门禁、灰度与回滚；最后用失败归因闭环连接线上反馈。

表达检查 · 不要只报框架或产品名；必须讲清输入输出、owner、证据、失败归因和为什么选择该模式。
