治理与可观测层理论课程

治理与可观测层（Governance & Observability）是贯穿在线实时请求的横切控制面：在输入、检索、模型决策、动作和输出边界执行策略，记录足以重放关键决定的 telemetry 与 audit evidence，并在高风险或异常时触发拒绝、降级、暂停和人工介入。

学习目标

  1. 解释在线治理为何是横切职责，而不是请求末尾的一次内容过滤。
  2. 用 Trace、Log、Metric 与 Audit 回答“发生了什么、为何发生、影响多大、谁批准”。
  3. 基于 subject、action、resource、context 设计最小权限与可追责授权链。
  4. 把 Prompt Injection、PII 与数据外泄防护放到输入、上下文、动作和输出多道门禁。
  5. 设计 HITL、预算护栏与事件响应闭环，明确暂停、恢复、遏制和复盘证据。

前置知识与学习顺序

前置为九层架构、编排控制、工具 / 动作层、知识 / RAG与记忆系统。推荐按顺序完成：

顺序单元必须产出
01在线横切职责与控制面门禁分布图与职责表
02Trace、Log 与 Metric遥测关联契约与诊断链
03身份、权限与审计授权输入、审批绑定与审计事件
04Prompt Injection、PII 与防护威胁模型与分层护栏
05HITL、预算与事件响应暂停恢复协议与事件闭环

组件边界

三处分工不可混用 · 在线治理与可观测在实时请求中强制策略并采集证据；离线安全评测在评测层消费 trace、运行回归与红队；沙箱执行隔离由模型网关 + 基础设施提供真实进程、网络、文件和凭据边界。三者协作，但不能互相替代。
相邻组件它负责本层负责 / 不负责
编排控制run 状态、路由、暂停与恢复本层返回 allow/deny/obligations；不拥有 run 状态机
工具 / 动作契约校验、幂等执行、类型化 observation本层授权动作并审计；不执行工具业务逻辑
评测层离线数据集、回归、judge 与红队本层提供线上证据与立即控制；不把线上告警当离线质量结论
模型网关 + 基础设施模型路由、运行时、凭据和沙箱隔离本层定义策略与预算；不宣称日志或 prompt 能提供真实隔离

完成标准

  • 能画出请求级门禁、遥测、审批和事件响应主链路。
  • 能为一次高风险工具调用提交可关联 trace、结构化日志、指标和审计证据。
  • 能准确说明在线治理、离线评测、沙箱隔离各自 owner 与接口。
  • 完成理论检查点：总分至少 80，且五项任务各至少 12/20。

available 只表示学习资产可用；是否掌握仍由进度跟踪中的提交证据和复测结果决定。
