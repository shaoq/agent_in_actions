04 Prompt Injection、PII 与防护

学习目标

  • 识别直接注入、间接注入、数据外泄与工具滥用路径。
  • 对 PII 做分类、最小化、目的限制、脱敏和保留控制。
  • 用分层防御与最小权限降低单一 guardrail 绕过后的影响。

核心概念

威胁路径关键控制
直接注入用户要求忽略策略指令优先级、结构化意图、动作强制授权
间接注入网页、RAG 文档、tool observation 携带恶意指令来源标记、数据隔离、最小上下文、工具 allowlist
PII 外泄prompt、trace、日志、输出或跨租户检索最小化、tokenization、ACL、redaction、retention
Exfiltration模型把秘密拼入 URL、邮件或工具参数egress allowlist、参数 schema、DLP、审批

工程要点

  1. 建立数据分类：public、internal、confidential、restricted/PII，并把分类标签随上下文和 trace 传播。
  2. 摄取前与检索前执行 ACL；检索内容标记 source 与 trust，绝不因进入 prompt 就升级为 instruction。
  3. 高风险动作在工具边界做确定性 schema、目的、资源、egress 和授权校验；模型分类器不能单独强制安全。
  4. 遥测默认 redact 或引用敏感载荷，访问审计与保留期独立；调试开关也不能绕过生产隐私策略。
  5. 把真实攻击 trace 脱敏后送往评测层形成离线安全回归；评测发现缺口后更新在线规则，但不在离线路径执行实时拦截。

常见坑

  • 追求“检测全部注入”：开放文本不存在可靠完美分类，应同时限制能力和数据范围。
  • PII 只做正则：上下文可能通过准标识符和组合信息重识别。
  • 输出过滤替代输入授权：秘密进入上下文后已扩大泄露面。
  • 把沙箱归给治理层：治理定义允许什么，基础设施提供真实文件、网络、进程和 secret 隔离。

面试表达

如何防 Prompt Injection？

我不承诺只靠提示词检测消灭注入，而是做纵深防御：外部内容带来源并始终作为数据；检索先授权，敏感字段最小化；模型只产候选意图，动作端再做确定性 schema、资源、egress 和权限校验；高风险操作需要审批，输出经过 DLP 与脱敏，运行时由基础设施沙箱隔离。真实攻击 trace 再进入离线安全回归。
