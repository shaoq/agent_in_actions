模型网关 + 基础设施理论课程

模型网关（Model Gateway）是推理内核访问模型的唯一受控入口，把厂商差异、能力匹配、路由、降级、限流、缓存、成本和延迟证据收敛为稳定契约；基础设施（Infrastructure）提供运行时、存储、网络、secret、资源配额以及工具执行所需的真实沙箱与租户隔离。两者共同回答“模型如何稳定可控地被使用，以及代码和数据在哪里以什么边界运行”。

学习目标

  1. 设计厂商无关的模型请求/响应契约，并按能力、风险、数据驻留和 SLO 路由。
  2. 用降级、限流、熔断与语义安全缓存处理依赖故障，不把业务错误伪装成成功。
  3. 把 token、延迟、并发和容量拆成可预算、可观测、可扩缩的工程量。
  4. 解释无状态计算、状态存储、队列和网络边界如何支撑可恢复运行。
  5. 设计租户隔离与工具沙箱，证明凭据、文件、网络和资源不会越界。

前置知识与学习顺序

前置为九层架构、推理内核、工具 / 动作层、治理与可观测和评测层。推荐按顺序完成：

顺序单元必须产出
01模型抽象、路由与能力匹配能力目录、统一契约与路由决策表
02降级、限流与缓存故障分类、降级矩阵与缓存键
03成本、延迟与容量预算分解、SLO 与容量模型
04运行时、存储与网络运行拓扑、状态归属与网络策略
05沙箱、租户与执行隔离隔离分层、威胁矩阵与销毁证据

组件边界

两条不可破坏的约束 · 推理内核必须统一经模型网关访问模型，不得在业务节点直接持有厂商 SDK 与密钥；治理层决定策略、工具层声明并发起动作，而基础设施实际提供可验证的进程、文件、网络、secret 和资源隔离。
相邻组件它负责本组件负责 / 不负责
推理内核形成结构化决策意图与不确定性信号提供统一模型能力；不决定业务下一步
编排控制run 状态、超时、重试、暂停恢复与终止报告调用证据和基础资源状态；不推进业务状态机
治理与可观测身份、策略、预算、审计和在线拦截强制执行传入的模型/网络/资源策略并产出遥测；不自创授权
工具 / 动作层工具契约、参数规范化、调用与 effect receipt提供实际沙箱、短期凭据和销毁；不拥有工具业务语义
评测层离线比较质量、成本、延迟与安全结果提供可重放环境和版本证据；不决定评测是否通过

完成标准

  • 能画出“推理内核 → 模型网关 → 厂商端点”的唯一访问链，并解释契约、能力与证据。
  • 能为超时、429、区域故障、预算耗尽和缓存命中设计不会越权的处理。
  • 能用延迟分位数、token/请求、并发、队列和配额完成容量估算。
  • 能证明工具隔离在身份、计算、文件、网络、secret、数据和生命周期七个维度成立。
  • 完成理论检查点：总分至少 80，且五项任务各至少 12/20。
