架构位置与边界

职责边界 · 模型网关拥有模型访问抽象、路由、韧性和用量证据；推理内核拥有决策语义。基础设施提供计算、存储、网络和隔离原语，但不拥有编排状态、记忆或知识的业务含义。

UNIT.03

模式与取舍

MODE 1

能力路由

先满足工具、多模态、区域和风险硬约束。

MODE 2

弹性降级

候选仍满足最低任务契约时切换。

MODE 3

安全缓存

确定性、无敏感泄漏且版本键完整时使用。

MODE 4

隔离执行

每租户/任务沙箱；成本更高但限制爆炸半径。

UNIT.05

生产工程约束

  • 统一错误分类区分限流、超时、内容拒绝和未知结果。
  • 路由记录候选、选择理由、端点版本和用量。
  • 容量观察 p95/p99、队列、token 速率、突发与上游配额。
  • 沙箱限制 CPU、内存、时间、文件、网络和秘密，并证明销毁。
UNIT.06

面试表达

30 秒回答

模型网关是推理内核访问模型的唯一平台边界，负责统一协议、能力路由、韧性和用量证据；基础设施提供运行、存储、网络与沙箱隔离，但不拥有上层状态和记忆的业务语义。

3 分钟回答结构

先讲统一请求与硬约束路由，再说明限流、降级和缓存必须保持任务契约；接着用成本、延迟、队列和 token 速率解释容量；最后讲租户、网络、凭据和资源隔离如何约束工具执行。

表达检查 · 不要只报框架或产品名；必须讲清输入输出、owner、证据、失败归因和为什么选择该模式。
