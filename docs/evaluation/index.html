评测层理论课程

评测层（Evaluation）在离线或受控环境消费 Agent 的最终结果与 trace，借助规则、人工和 LLM-as-a-Judge 形成可比较的质量证据，再把失败切片、门禁结论和修复建议反馈给编排及相应组件 owner。它回答“系统整体是否达标、变更是否可发布、哪类失败应修在哪里”。

学习目标

  1. 从产品目标、风险和失败分类构建版本化目标数据集与可重复基线。
  2. 同时评价最终结果与执行轨迹，避免“答案碰巧正确”掩盖危险路径。
  3. 按确定性、成本与风险选择规则、人工或 Judge，并校准偏差与漂移。
  4. 把统计比较、关键切片和安全约束转成可执行回归门禁。
  5. 把安全红队与脱敏线上 trace 沉淀为新样本，形成“发现—评测—反馈—修复—复测”闭环。

前置知识与学习顺序

前置为九层架构、编排控制和治理与可观测。推荐按顺序完成：

顺序单元必须产出
01目标、数据集与基线失败分类、数据卡与基线报告
02结果评测与轨迹评测双层指标与 trace 评分契约
03规则、人工与 LLM-as-a-Judge评分路由、rubric 与校准方案
04回归测试与上线门禁比较协议与发布决策矩阵
05安全红队与线上反馈威胁覆盖与反馈闭环

组件边界

离线评测 ≠ 在线治理 · 评测层消费已产生的结果与 trace，形成聚合结论并反馈编排策略；治理与可观测层在实时请求中执行授权、预算、告警和拦截。离线 Judge 不能替代在线门禁，在线告警也不能直接当作离线质量结论。
相邻组件它负责评测层负责 / 不负责
编排控制run 状态、路由、重试、暂停恢复与策略执行反馈失败切片和策略建议；不直接推进 run 生命周期
治理与可观测在线策略强制、telemetry、audit 与事件响应离线消费脱敏 trace、验证规则效果；不执行实时拦截
推理 / 工具 / RAG / 记忆各自生产决策、动作、知识和经历用指标归因到正确 owner；不复制各层运行逻辑
模型网关 + 基础设施模型路由、隔离执行和评测运行环境定义实验协议与验收；不提供真实沙箱边界

完成标准

  • 能为同一 Agent 定义目标、数据版本、基线、结果指标、轨迹指标和关键切片。
  • 能解释规则、人工与 Judge 的选择、校准、盲评、偏差和成本取舍。
  • 能给出带统计不确定性、关键安全约束和回滚条件的发布门禁。
  • 完成理论检查点：总分至少 80，且五项任务各至少 12/20。

available 只表示资产可用；是否掌握仍由进度跟踪中的证据与复测结果决定。
