Agent 应用工程师理论优先学习路线

本路线服务于两个目标:先系统理解 Agent 应用工程,再把理解转化为实现、调试、考核和面试能力。时间只用于安排节奏,阶段是否完成由能力证据决定。

两大阶段

第一阶段:学习与训练

学习阶段按 L0 到 L4 逐层推进:

层级 目标 主要产出 进入下一层条件
L0 Agent 基础 建立共同术语,区分 Agent、Workflow、Chatbot 和 LLM Application 术语表、场景分类、基础概念图 通过 Agent 基础理论检查点
L1 九层知识地图 理解九层职责、上下游、在线主链路和离线旁路 九层图、请求级数据流、边界对照表 通过 L1 九层 Agent 架构知识地图检查点
L2 组件理论深潜 逐层理解机制、模式、取舍和失败治理 专题笔记、模式矩阵、失败矩阵、理论追问 通过对应组件无代码理论门禁
L3 工程验证 用最小实验验证理论,训练 trace 调试 可运行实验、测试、失败复盘 能实现、解释并诊断主要失败路径
L4 面试与综合设计 将理论和实践转成结构化表达 高频问答、系统设计、项目复盘 通过模块复测后进入综合考核

当前处于 L0 Agent 基础。现有编排实验已经可用,但推荐在 L2 编排理论门禁通过后再运行。

第二阶段:考核与查漏补缺

考核阶段不再输入新知识,而是验证迁移能力:

  1. 模块复测:使用新案例检查是否仍然掌握。
  2. 限时系统设计:覆盖主流程、失败、权限、安全、成本和评测。
  3. 限时 trace 调试:基于证据定位模型、检索、工具、状态、权限或评测问题。
  4. 端到端项目:实现包含工具、RAG、记忆、评测和 trace 的最小系统。
  5. 模拟面试:连续追问架构选择、工程取舍、失败经验和上线策略。

毕业标准:

  • 能独立设计生产级 Agent 应用的端到端架构。
  • 能实现核心链路并处理失败路径。
  • 能基于 trace 做分层归因,而不是只改 prompt 试错。
  • 能解释为何拒绝某个复杂方案。
  • 能把理论、代码和失败案例组织成可信的面试回答。

三轮学习方法

第一轮:横向全景

先学习 Agent 基础与九层地图,不写代码。目标是知道每个概念位于哪里,避免在缺少全局坐标时陷入框架 API。

第二轮:纵向理论深潜

每个组件统一回答六类问题:

  1. 定义与动机:它是什么,为什么存在?
  2. 架构边界:负责什么,明确不负责什么?
  3. 内部机制:有哪些子能力,数据如何流动?
  4. 方案模式:常见模式和替代方案是什么?
  5. 工程取舍:如何根据不确定性、成本、延迟、风险和可审计性选择?
  6. 失败与治理:如何失败,如何观测、恢复、评测和约束?

每个理论专题必须包含概念图、流程或时序、方案对比矩阵、失败矩阵、场景题、理论追问和下一步。

第三轮:工程验证

理论门禁通过后再进入实验。实验用于证明或修正理解,不是第一次接触概念。每次实验都要回答:

  • 哪个理论概念在代码中由什么结构表达?
  • 正常路径和失败路径分别是什么?
  • trace 提供了哪些证据,又缺少哪些证据?
  • 当前实现是教学简化、项目推荐还是生产通用做法?

理论门禁

阅读完成不等于理论通过。进入某组件实验前,至少提交:

  1. 一份脱稿定义和边界说明。
  2. 一张组件上下游与数据流图。
  3. 一份主要模式对比和场景选择题。
  4. 一份失败矩阵或 trace 归因记录。
  5. 一次至少两轮追问的口述记录。
  6. 一道无代码系统设计题。

只有在上述证据达到对应检查点标准后,进度才从“理论学习中”变为“理论通过 / 可进入实验”。超过建议时间但证据不足时,应补强具体缺口,不能按日历自动进入 L3。

当前学习顺序

  1. 阅读Agent 基础理论。
  2. 完成Agent 基础理论检查点。
  3. 学习并复述九层架构总览。
  4. 完成L1 九层架构知识地图检查点。
  5. 通过后进入编排理论课程。
  6. 完成编排无代码理论检查点。
  7. 通过后进入推理内核四单元理论课程。
  8. 完成推理内核理论检查点，总分至少 80 且每项至少达到 60%。
  9. 通过后进入工具 / 动作层五单元课程，再完成工具理论检查点，总分至少 80 且每项至少达到 60%。
  10. 工具理论通过后进入知识 / RAG 五单元课程，再完成知识 / RAG 理论检查点，总分至少 80 且每项至少达到 60%。
  11. 在进度页登记真实门禁证据后，再按对应路线进入实验或下一组件；编排实验入口仍为最小 Agent Loop。

防止理论学习跑偏

  • 不以阅读数量、收藏链接或学习天数作为完成标准。
  • 不追求穷尽所有框架和论文,优先掌握稳定概念与工程边界。
  • 不把单一厂商、框架或项目方案写成行业唯一标准。
  • 不无限推迟实践;理论门禁通过后必须进入对应实验验证。
  • 不提前学习 Multi-Agent 实现,先完成单 Agent 编排理论。
